有些同学平时私信我有木有好的前端学习资料推荐，确实正确的资料是灰常重要滴，傲天当初就是分不清Java和javascript才上了前端这条海盗船啊~~

  1. 前端技能汇总 Frontend Knowledge Structure

    https://github.com/JacksonTian/fks 

    评价：阿里巴巴大神JacksonTian的Project，在GitHub拥有1w+星星，readme写的不是很专业，但是把前端知识有条理地梳理成十几个部分。从基础web技术和CS理论、到性能优化、到企业化打包部署，可谓是从入门到精通它都有。所以它的缺点也很明显，某个点或者模块不够深入不够专业。

  2. 猫的前端回忆录

    https://github.com/windiest/Front-end-tutorial 

    windiest（国服最渣切图妹）的作品，她本人专注于写各种教程，而该教程是她最受欢迎的一个，拥有2.8k个星星。特点是专注于前端JavaScript，CSS，HTML技术。但是并不是基础知识，而是深入编程语言的某个点（比如JS深浅拷贝）或者技术实践的某个点（比如JSONP原理）。同时对于Angular、vue、react这三个目前（2017年7月）最火的前端框架分别深入介绍。

    特点是介绍任何技术都是深入某个点，所以不适合刚学习某项技术的同学。

  3. 没事写写文章，喜欢的话请点star，想订阅点watch，千万别fork！

    https://github.com/fouber/blog 

    张云龙（UC工程师）前辈作品，特点如题，只是写写文章，但是却有1.2w个星星，无敌！作者以时间为轨迹，记录了自己开发的点点滴滴，并分享自己开发过程中遇到的一些坑和挖坑的经历，以及挖坑后的思考，作者把最真实的自己展现给读者，可谓是良心作品。适合于要成为大牛的读者，看看大神的心路历程总是有帮助的。缺点嘛，那就是作者想到啥写啥，没有什么知识体系、模块分类、哈哈、、、

  4.  A collection of awesome browser-side JavaScript libraries, resources and shiny things.

    https://github.com/sorrycc/awesome-javascript 

    https://github.com/jobbole/awesome-javascript-cn （中文翻译版）

    很骚的JavaScript，是的，你想写出骚骚的JS代码么？那么来看看这个拥有9.7K星星的Project吧。项目专注于介绍Javascript。（是awesome系列项目之一，不知道开源教程awesome系列的同学可以搜索一下，很有名）。该项目特点就是专注于JS开发，从前端到后台（nodejs），内容专业，分类清晰！！它把JS开发细分成很多个知识模块，例如：包管理，模块加载，打包部署，测试框架，MVC框架，基于Node技术的CMS系统，模板引擎，数据可视化工具等等，你不是需要全部学会这些，这些对于读者学习JS是一个方向灯塔，告诉我们应该学习什么。缺点就是不介绍JS基础编程知识，额，你一个专注于JS的教程项目应该给个基础语法学习的版块吧？可惜它木有。。

所以介绍了这么多，就是没有一个介绍一个基础语法的项目了？如果酱紫，读者肯定是不满意的。我们每个开发者都是从新手走过来，而新手最主要的问题是不懂基础语法。所以这里强烈推荐W3 School //www.w3school.com.cn/ 笔者大学时候最喜欢的前端基础知识学习网站，免费没毛病~~


写在后面，笔者学习编程5年，从事开发工作2年，不自量力总结软件开发工程师最重要的四点特征：（大神轻喷(>_<)

  1. 基础语法  （你搞JS开发问你数组操作的几个函数都讲不出来不像话吧，你搞Java写个简单的IO，写个注解解释器都搞不定也说不过去吧）

  2. 基础算法与数据结构   （清楚链表与哈希表等简单数据结构以及相应的操作时间复杂度是非常重要的，因为平时这与大家平时开发都是息息相关的，即使我们大部分人是偏向于“应用”（非研究类型）的工程师，因为我经常看到一些同事写出O(N^2)复杂度的代码，而往往这些地方组合使用链表与哈希表就能把时间复杂度降到O(N)，以空间换时间。所以笔者对于算法的态度是：不搞太深，但基础的一些都会。我当初dijkstra与MD5写的飞起，现在都忘了，但让我手写快排1分钟搞定）

  3. 好的代码结构与注释以及最佳实践（Best Practice）  （是的，大学的时候我们代码写的飞起，洋洋洒洒，别人都看不懂，以为很酷~~但这是团队协作大忌，统一的代码结构与良好的注释是一个工程师非常重要的品质，如果你写的时候没时间写注释，可以在以后几天抽个时间统一补上也可以）

  4. 分享 （你有什么好的想法尽量分享给大家，我知道这样很难，因为以前我们大学心理学教授说：程序员之所以沉默寡言，是因为他们长时间与没有生命的事物打交道（指电脑）。所以多参加一些线下技术分享会，多于他人交流。Google在各大城市都有线下分享会（GDG）定期举行，有免费小吃与饮料。）
