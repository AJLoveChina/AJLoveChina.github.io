我们都知道在web处理json的最主要俩个函数分别是：

  1. JSON.stringify    --序列化json对象，生成json字符串

  2. JSON.parse    --解析json字符串，生成json对象

这俩个函数兼容性非常好，但是在IE7-（包括IE7）不支持。

当然啦，截止2017.6.4，不兼容IE7已经不是什么大问题，好多新的web应用直接放弃不支持HTML5的浏览器，强势！web工程师看到了黎明的曙光。

群众：JSON.stringify除了序列化JSON以外，还有什么隐藏的功能吗？

AJ：那就是以漂亮的方式序列化JSON

群众：what the fake?

看下面代码：

//我们定义一个变量
var obj = {name:"hejie",age:12,marry:false};

如果你简单地调用stringify方法： 

JSON.stringify(obj)

输出 

{"name":"hejie","age":12,"marry":false}

但是如果你按下面这么调用： 

JSON.stringify(obj, null, 4)

输出 

{
    "name": "hejie",
    "age": 12,
    "marry": false
}

Try it 使用jsfiddle试试

PS：第三个参数4指的是使用4个空格格式化层级，你可以使用其它数值或者制表符 \t

PPS：支持这俩个函数的浏览器都支持这么格式化的做法，而且json2.js中也有相应的实现，意思就是低版本浏览器中如果你使用了json2也可以这样美化json。

然后， 我猜你想知道着第二个参数null到底是做什么的，下面这段摘自MDN，数据专业，童叟无欺。

语法 

JSON.stringify(value[, replacer [, space]])

参数

  • value

  • 将要序列化成 一个JSON 字符串的值。

  • replacer 可选

  • 如果该参数是一个函数，则在序列化过程中，被序列化的值的每个属性都会经过该函数的转换和处理；如果该参数是一个数组，则只有包含在这个数组中的属性名才会被序列化到最终的 JSON 字符串中；如果该参数为null或者未提供，则对象所有的属性都会被序列化；关于该参数更详细的解释和示例，请参考使用原生的 JSON 对象一文。

  • space 可选

  • 指定缩进用的空白字符串，用于美化输出（pretty-print）；如果参数是个数字，它代表有多少的空格；上限为10。该值若小于1，则意味着没有空格；如果该参数为字符串(字符串的前十个字母)，该字符串将被作为空格；如果该参数没有提供（或者为null）将没有空格。
