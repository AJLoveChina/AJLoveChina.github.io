在web前端，时间处理一直是个令人头疼的事情，原生的做法有以下缺点：

  1. 兼容性。各浏览器Date类实现的差异

  2. 原生Date类提供的功能太少，很多功能需要自己实现，比如时间格式化等.


    而moment.js就是为了解决这样的问题而诞生的，你不需要记住它全部的API，这里我列举了最常用、最实用的一些方法，只要掌握它们，前端时间处理游刃有余！（题外话：momentjs不依赖任何第三方库，网页和nodejs中都可以使用，但是网页或者nodejs中需要使用对应的momentjs）

$.最佳实践

如果你的前端项目使用了momentjs, 任何时候你都应该尽量操作moment对象, 而不是原生的Date对象, Date对象可以转换成moment对象.


$.moment 对象与 Date对象的相互转换 

moment("2017-02-02").toDate(); // moment 对象转换成  Date 对象

moment(new Date()); // Date 对象转换成 moment对象

$.时间格式化与解析

1.格式化当前时间，每次调用获取的都是系统的当前时间 

// 输出类似这样 ： 2017/09/22 09:38:39
// moment() 函数等同于  moment(new Date())
moment().format('YYYY/MM/DD HH:mm:ss')

2.格式化特定时间
 

moment('2017-3-09 15:31:23', 'YYYY-MM-DD HH:mm:ss');


$.获取/设置 时间的年, 月, 日, 小时, 分钟, 秒 

// 获取
moment().seconds(); // 秒
moment().minute();  // 分钟
moment().hour();    // 小时
moment().date();    // 日(日期,例如7月24号的 24)
moment().month();   // 月 (注意:月份 0表示1月份,1 表示2月份,类推. 只有月份如此, 日期等其他都不是酱紫.别问为啥子, JS Date对象也是这样)
moment().year();    // 年

moment().day();     // 这个表示星期几

// 设置 设置时间很简单, 只要在上述的方法中传入参数即可,我举俩个例子
moment().hour(14); // 下午俩点
moment().month(9); // 十月份,别搞错了. 月份特殊!!
// 关于设置时间我多说一句, 就是比如你把月份从9月改成了8月,那么对应的日期等信息都是可能发生变化的, 意思就是momentjs不是简单的把数字改改而已


$.时间比较 

// 1.isBefore  是否在某个时间之前
moment("2017-12-12").isBefore("2017-12-13") // true
// 2.isAfter 是否在某个时间之后
moment("2017-12-12").isAfter("2017-12-13"); // false
// 3.isBetween 是否在某个时间段之间
moment('2010-10-20').isBetween('2010-10-19', '2010-10-25'); // true


$.时间偏移 

// 所有时间偏移的方法都返回  moment 对象
moment().add(1, "years"); // 获取一年后的Date对象
moment().add(2, "months");// 俩个月后
moment().add(3, "days");
moment().add(4, "hours");
moment().add(5, "minutes");
moment().add(6, "seconds");
moment().add(1, "days").add(2, "hours"); // 支持级联操作

$.题外话

  2. moment自定义月份, 星期等显示名称
