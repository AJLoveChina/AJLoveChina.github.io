首先说明一下，nrm（外号牛肉面）不是我开发的，只是我平时觉得非常好用的工具想推荐给大家。

本文面向读者：喜欢发布项目到npm官网的童鞋。

-----------------手动分割线-------------------------

喜欢使用npm的童鞋都知道，npm官网下载包的速度非常之慢，加上npm不支持断点续传，所以经常卡到安装依赖失败（执行npm install失败）。

所以目前一般有2个做法：

  1. 使用类似cnpm这样的工具， 它由淘宝团队开发， 几乎适配所有npm命令（例如 ： cnpm install）

  2. 使用`npm config set registry 镜像地址`来切换到镜像地址，提高下载包的速度


这俩种做法如何做抉择呢？

因为像cnpm这样替代npm的工具，往往下载的包附带有一些特殊标志。这会导致你不能修改项目所在的磁盘父级目录。也就是说你不能把这个项目从一个目录拷贝到另一个目录，否则移动目录之后运行项目会出现`依赖找不到`这样的异常错误。

当然，如果你说：我能接受删除node_modules重新安装！那么你选择哪种方案无所谓啦，开心就好~~


如果你有一种执念，我就要使用npm命令，不要其它替代的工具。

再加上你还喜欢发布项目到npm官网的话，你肯定和我一样为一件事发愁，我记不住npm官方包的地址啊！如下：

npm config set registry https://registry.npmjs.org/

那么！！你就需要今天老哥给你推荐的牛x工具了 ： nrm
